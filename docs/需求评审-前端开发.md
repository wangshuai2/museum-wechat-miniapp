### 1.2 核心页面设计要点

#### 博物馆列表页
- **搜索栏**: 支持搜索 + 城市选择
- **筛选栏**: 类型、距离、评分
- **列表项**: 图片、名称、城市、评分、距离
- **下拉刷新**: 更新数据
- **上拉加载**: 分页加载

#### 博物馆详情页
- **头部**: 封面图、名称、收藏按钮
- **信息栏**: 地址、开放时间、门票
- **Tab 切换**: 简介/展品/攻略/评价
- **镇馆之宝**: 横向滑动卡片
- **底部操作**: 写笔记、记录参观

#### 足迹记录页
- **快速记录**: 搜索选择博物馆
- **日期选择**: 日历组件
- **照片上传**: 最多9张
- **心情标签**: 开心/震撼/学到很多...
- **参观评分**: 星级评分

#### 足迹地图页
- **地图展示**: 小程序地图组件
- **城市标记**: 参观过的城市
- **聚合显示**: 密集区域聚合
- **点击交互**: 显示城市详情

## 三、交互设计建议

### 3.1 关键交互点

#### 参观记录流程
```
点击"记录参观" → 搜索/选择博物馆 → 选择日期 → 
上传照片（可选） → 填写笔记（可选） → 评分 → 保存
```

**优化建议**:
- 支持快速记录（只选博物馆和日期）
- 提供模板引导填写
- 支持从相册批量导入

#### 足迹地图交互
```
进入地图 → 显示城市标记 → 点击城市 → 弹出城市详情 → 
点击查看 → 显示该城市的博物馆列表
```

**优化建议**:
- 首次进入动画引导
- 支持缩放和拖拽
- 未参观城市灰色显示

### 3.2 微交互建议

| 场景 | 交互 |
|------|------|
| 记录成功 | 烟花动画 + "记录成功"提示 |
| 收藏博物馆 | 心形动画 + "已收藏"提示 |
| 达成成就 | 徽章弹出动画 |
| 首次参观 | 欢迎动画引导 |
| 年度总结 | H5 动画展示 |

## 四、UI 组件库选择

### 4.1 推荐方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **TDesign** | 腾讯官方、设计规范好 | 组件相对较少 |
| Vant Weapp | 社区活跃、组件丰富 | 需要二次封装 |
| 自研 | 完全可控 | 开发成本高 |

**建议**: 使用 **TDesign** 作为基础组件库，自定义业务组件

### 4.2 需要自定义的组件

- 博物馆卡片样式
- 足迹地图组件
- 时间轴组件
- 统计图表组件
- 笔记编辑器

  this.setData({ loading: true });

  this.setData({

### 5.3 数据缓存

```javascript
// 本地缓存策略
const CACHE_KEY = 'museum_list';
const CACHE_EXPIRE = 30 * 60 * 1000; // 30分钟

async function getMuseumList() {
  const cached = wx.getStorageSync(CACHE_KEY);
  if (cached && Date.now() - cached.timestamp < CACHE_EXPIRE) {
    return cached.data;
  }
  const data = await fetchMuseumList();
  wx.setStorageSync(CACHE_KEY, { data, timestamp: Date.now() });
  return data;
}
```
