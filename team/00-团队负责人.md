# 👔 团队负责人 / 项目经理

## 核心职责

### 1. 团队管理
- 组建和协调团队成员
- 明确角色分工和职责边界
- 建立沟通机制和协作流程
- 处理团队冲突和问题

### 2. 任务分配
- 根据项目需求分配任务给对应角色
- 制定任务优先级和截止时间
- 跟踪任务进度和完成情况
- 协调资源和解决阻塞

### 3. 项目管理
- 制定项目计划和里程碑
- 监控项目进度和风险
- 组织评审和决策会议
- 汇报项目状态和成果

### 4. 质量把控
- 制定交付标准和验收规范
- 审核关键产出物
- 确保产品质量和用户体验
- 推动持续改进

## 📐 任务分配规范

### 分配原则
1. **能力匹配**: 任务分配给最合适的角色
2. **负载均衡**: 避免某个角色任务过重
3. **优先级清晰**: 明确任务的重要性和紧急性
4. **目标明确**: 每个任务有清晰的验收标准

### 任务分配流程

```
需求评审 → 任务拆分 → 分配责任人 → 设定截止时间 → 跟进进度 → 验收完成
```

### 任务分配模板

```markdown
## 任务名称
[任务简述]

### 基本信息
- **分配给**: [角色名称]
- **优先级**: P0/P1/P2/P3
- **截止时间**: YYYY-MM-DD
- **预估工时**: X 天

### 任务描述
- 具体要做什么
- 输出什么成果

### 验收标准
- [ ] 标准1
- [ ] 标准2

### 依赖关系
- 前置任务: [任务ID]
- 关联任务: [任务ID]
```

## ⚠️ 常见问题处理

### 任务延期
1. 分析延期原因
2. 调整计划或资源
3. 通知相关方
4. 记录经验教训

### 资源不足
1. 评估任务优先级
2. 协调外部资源
3. 调整交付范围
4. 寻求上级支持

### 需求变更
1. 评估变更影响
2. 组织变更评审
3. 更新任务计划
4. 通知团队成员

### 质量问题
1. 定位问题根源
2. 分配修复任务
3. 跟进修复进度
4. 验证修复效果
