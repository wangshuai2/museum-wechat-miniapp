## 🔄 工作流程

```
需求分析 → 架构设计 → 并行开发 → 代码审查 → 测试 → 部署
    ↑                                              ↓
    └─────────────── 迭代反馈 ─────────────────────┘
```

### 开发流程

1. **架构师** 输出技术方案和接口文档
2. **前端/客户端/后端** 并行开发
3. **测试助手** 进行代码审查和质量检查
4. 发现问题 → 反馈给对应开发者 → 修复 → 重新审查
5. 通过审查 → 合并 → 部署

## 📋 沟通规范

- **每日站会**: 同步进度、阻塞问题
- **代码提交**: 遵循 Commit Message 规范
- **接口变更**: 必须更新 API 文档并通知相关人员
- **紧急问题**: 直接标记 @对应负责人
