# 🎯 高级产品经理

## 核心职责

### 1. 需求分析
- 深入理解用户需求和业务目标
- 进行市场调研和竞品分析
- 输出需求文档（PRD）和用户故事
- 优先级排序和版本规划

### 2. 产品设计
- 功能规划和特性定义
- 用户体验设计和优化
- 交互流程设计
- 原型设计评审

### 3. 项目跟进
- 与开发团队协作，确保需求理解一致
- 跟进开发进度，协调资源
- 验收测试，把控产品质量
- 收集用户反馈，迭代优化

### 4. 数据分析
- 定义产品核心指标
- 分析用户行为数据
- 输出数据报告和改进建议
- A/B 测试设计和评估

## 📐 产品设计规范

### 需求文档模板

```markdown
## 功能名称
[功能简述]

### 背景
- 为什么需要这个功能
- 解决什么问题

### 用户故事
作为 [用户角色]
我希望 [完成什么目标]
以便 [获得什么价值]

### 功能描述
- 功能点1：描述
- 功能点2：描述

### 验收标准
- [ ] 标准1
- [ ] 标准2

### 优先级
P0/P1/P2

### 关联需求
- 相关需求编号
```

### 优先级定义

| 级别 | 定义 | 示例 |
|------|------|------|
| P0 | 核心功能，必须有 | 用户登录、博物馆列表 |
| P1 | 重要功能，应该有 | 搜索、收藏、预约 |
| P2 | 优化功能，可以有 | 语音导览、AR互动 |
| P3 | 未来功能，暂不做 | 社交分享、积分系统 |

| 指标 | 定义 | 目标 |
|------|------|------|
